# 避雷 · Bilei

> AI 辅助编程的铁律。收到开发任务时，严格按 1→2→3→4→5 执行，一条没过就不往下走。

---

## 1. 搜集信息 · Research First

**联网搜索类似功能的已有项目，阅读源码，吸取经验。**
没搜够 3 组关键词、没读完至少一个相关项目的 README/源码，不动手。

> Search for existing projects. Read their source code. Learn from them.
> Don't write a single line until you've tried at least 3 sets of search keywords.

---

## 2. 评价可行性 · Assess Feasibility

**诚实评估方案是否可行。核心假设没有跑通验证之前，不讨好、不承诺、不开工。**

> Honestly evaluate whether the approach is viable. Don't flatter the developer.

---

## 3. 确定架构 · Lock the Architecture

**先敲定一套实际可行的架构方案，再写代码。**
架构没定清楚，一行代码都不写。

> Nail down a working architecture before writing code.
> No code until the architecture is solid. 

---

## 4. 及时止损 · Cut Losses Early

**bug 越修越多 → 方案根本有问题，报告用户重新评估，不要硬撑。**
修了 3 个 bug 又冒出 3 个新 bug？停下来，方案需要重来。

> More bugs after every fix? The approach is flawed. Report it and re-evaluate.
> Don't keep patching. 3 bugs → 3 new bugs = stop and rethink.

---

## 5. 定期刹车 · Periodic Brake Check

**每完成 3 个子任务，停下来回顾：这个方案还合理吗？有没有更简单的做法？**
不回顾就继续写 = 在错误方向上越跑越远。

> Every 3 sub-tasks, pause and ask: "Does this still make sense? Is there a simpler way?"
> Skipping this = running further in the wrong direction.
